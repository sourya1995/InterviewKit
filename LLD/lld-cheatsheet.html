Interview Questions

The First 10 Minutes — Questions to Ask

Strategy
01
Functional Scope
  • What are the core use cases we must support?
  • What is explicitly out of scope?
  • Is this a single-tenant or multi-tenant system?
  • What are the primary user roles/actors?
02
Entities & Relationships
  • What are the main nouns / domain objects?
  • What are the key 1:1, 1:N, M:N relationships?
  • Are there any lifecycle states we need to track?
  • Is there a concept of membership, roles, or ownership?
03
Behavioral Expectations
  • Walk me through the happy-path flow end-to-end.
  • Who triggers each major action — user, timer, system?
  • What side effects happen when action X occurs?
  • Are notifications or events emitted to other systems?
04
Edge Cases & Constraints
  • What happens when capacity is full or a resource is unavailable?
  • How do we handle concurrent modifications to the same entity?
  • Are there reservation vs. walk-in style conflicts?
  • What is the cancellation / rollback policy?
05
Non-Functional Requirements
  • What is the expected read vs. write ratio?
  • Do we need real-time updates or is polling acceptable?
  • Are there SLA or latency requirements?
  • Is high availability a concern, or just correctness?
06
Extensibility Intent
  • What features are most likely to be added in v2?
  • Should pricing/rules be configurable without code changes?
  • Will new entity types need to be introduced (e.g., new vehicle type)?
  • Is plugin or third-party integration a future concern?

Design Pattern Quick Reference

GoF Patterns
Creational Patterns
Singleton
One instance globally. Use for shared resources like a logger, config, or connection pool.
Factory Method
Subclasses decide which class to instantiate. Decouples creation from usage.
Abstract Factory
Creates families of related objects without specifying concrete classes.
Builder
Construct complex objects step-by-step. Useful when many optional fields exist.
Prototype
Clone existing objects rather than creating from scratch. Good for expensive init.
Object Pool
Reuse pre-instantiated objects. Critical for thread, connection, or locker pools.
Structural Patterns
Adapter
Bridges incompatible interfaces. Wrap legacy or third-party code.
Decorator
Add behavior dynamically without subclassing. Layered feature additions.
Facade
Simplified interface to a subsystem. Hide complexity behind a clean API.
Composite
Tree structures of objects treated uniformly. File systems, menus, org charts.
Proxy
Surrogate for another object. Lazy loading, access control, caching.
Bridge
Decouple abstraction from implementation. Change both independently.
Flyweight
Share fine-grained objects to reduce memory. Chess pieces, map tiles.
Behavioral Patterns
Observer
One-to-many event notifications. Publish/subscribe, UI events, stock tickers.
Strategy
Swap algorithms at runtime. Pricing rules, sort strategies, payment methods.
State
Object changes behavior when its internal state changes. ATM, vending machine.
Command
Encapsulate requests as objects. Undo/redo, queuing, logging of actions.
Iterator
Sequentially access elements without exposing internals.
Template Method
Define algorithm skeleton in base class; subclasses fill in steps.
Chain of Responsibility
Pass request along a chain of handlers. Middleware, logging pipelines.
Mediator
Centralize complex communications between objects. Chat rooms, ATCs.

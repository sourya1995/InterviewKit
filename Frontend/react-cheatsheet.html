01 Debouncing & Throttling Performance
Dating analogy:
🔵 Debounce — You're drafting a text to your crush. You keep editing it every few seconds. The text only sends after you stop typing for 1 second. No embarrassing half-finished messages.

🟠 Throttle — You want to call her repeatedly, but your phone enforces a rule: max 1 call every 2 seconds, no matter how many times you tap "Call".
Debounce — wait until user stops typing
import { useState, useRef } from 'react';

function SearchBox() {
  const [result, setResult] = useState('');
  const timer = useRef(null);           // holds the timer between renders

  const handleChange = (e) => {
    clearTimeout(timer.current);          // cancel previous timer on every keystroke
    timer.current = setTimeout(() => {
      setResult(`Searching for: ${e.target.value}`); // fires 1s after LAST keystroke
    }, 1000);
  };

  return (
    <div>
      <input onChange={handleChange} placeholder="Type to search..." />
      <p>{result}</p>
    </div>
  );
}
Throttle — max once per interval
function LikeButton() {
  const lastFired = useRef(0);            // timestamp of last call

  const handleLike = () => {
    const now = Date.now();
    if (now - lastFired.current < 2000) return; // block if < 2s since last call
    lastFired.current = now;
    console.log('💕 Liked!');              // only runs once every 2 seconds
  };

  return <button onClick={handleLike}>❤️ Like</button>;
}
02 useLayoutEffect Hook
useEffect runs after the browser paints — the user briefly sees the old state.
useLayoutEffect runs before the browser paints — you measure/fix the DOM first, user sees nothing weird.
Rule of thumb: Use useEffect always. Switch to useLayoutEffect only when you see a visual flicker. 
import { useRef, useState, useLayoutEffect } from 'react';

function Tooltip() {
  const boxRef = useRef(null);
  const [left, setLeft] = useState(0);

  useLayoutEffect(() => {
    // DOM is ready but browser hasn't painted yet
    // Safe to read size/position and adjust BEFORE user sees anything
    const width = boxRef.current.getBoundingClientRect().width;
    setLeft(-width / 2);                  // center the tooltip
  }, []);

  return (
    <div ref={boxRef} style={{ position: 'absolute', left }}>
      I'm perfectly centered 🎯
    </div>
  );
}

// useEffect  → runs AFTER paint → may flicker
// useLayoutEffect → runs BEFORE paint → no flicker
03 position: relative / absolute / z-index CSS
position: relative — "I stay in the normal flow, but I can nudge myself."
position: absolute — "I escape the normal flow and position myself relative to my nearest relative parent."
z-index — who sits on top. Higher number = in front. Only works on positioned elements. 
/* THE GOLDEN RULE:
   absolute child needs a relative parent, otherwise
   it positions itself relative to the whole page */

.parent {
  position: relative;   /* 🔑 anchor point for absolute children */
  width: 300px;
  height: 200px;
  background: #1e293b;
}

.badge {
  position: absolute;   /* escapes normal flow */
  top: 8px;            /* 8px from top of .parent */
  right: 8px;           /* 8px from right of .parent */
  z-index: 10;          /* sits on top of siblings */
}

.overlay {
  position: absolute;
  inset: 0;             /* top/right/bottom/left all = 0 → covers parent fully */
  z-index: 5;           /* below badge (10) but above normal content */
}

/* --- JSX --- */
function Card() {
  return (
    <div className="parent">
      <p>Normal content (in flow)</p>
      <div className="overlay">Dim overlay</div>
      <span className="badge">NEW</span>   {/* sits on top */}
    </div>
  );
}
04 Universal Error Handler (Error Boundary) Resilience
Wrap your whole app in an Error Boundary — if any child crashes, it catches the error and shows a fallback UI instead of a blank white screen. Like a try/catch, but for your component tree. 
import { Component } from 'react';

// Must be a class component — no hook equivalent exists yet
class ErrorBoundary extends Component {
  state = { crashed: false, error: null };

  // React calls this when a child throws
  static getDerivedStateFromError(error) {
    return { crashed: true, error };
  }

  componentDidCatch(error, info) {
    console.error('💥 App crashed:', error, info); // send to Sentry etc.
  }

  render() {
    if (this.state.crashed) {
      return <h2>Something went wrong 😵</h2>;
    }
    return this.props.children; // render normally if no crash
  }
}

// Wrap your app once — catches ALL errors inside
function App() {
  return (
    <ErrorBoundary>
      <MyApp />          {/* any crash here is caught */}
    </ErrorBoundary>
  );
}
05 Data Provider with Context Performance
Without context, you pass data down through every parent → child → grandchild (prop drilling). Context is a broadcast — any component anywhere can tune in. Splitting into separate contexts avoids re-rendering unrelated components. 
import { createContext, useContext, useState } from 'react';

// 1. Create the context
const UserContext = createContext(null);

// 2. Provider — wraps the tree, holds the data
function UserProvider({ children }) {
  const [user, setUser] = useState({ name: 'Priya', role: 'admin' });

  return (
    <UserContext.Provider value={{ user, setUser }}>
      {children}   {/* only re-renders when user state changes */}
    </UserContext.Provider>
  );
}

// 3. Any child reads it — no prop drilling needed
function Navbar() {
  const { user } = useContext(UserContext);
  return <p>Hello, {user.name}!</p>;
}

// 4. Use provider at the top level
function App() {
  return (
    <UserProvider>
      <Navbar />        {/* no props needed! */}
      <Dashboard />
    </UserProvider>
  );
}
06 Suspense Loading UI
Suspense says: "While this component is loading (lazy import or async data), show this fallback UI instead of a blank screen." Think of it as a built-in loading boundary. 
import { Suspense, lazy } from 'react';

// lazy() = don't load this file until it's actually needed
const HeavyChart = lazy(() => import('./HeavyChart'));

function Dashboard() {
  return (
    <Suspense fallback={<p>Loading chart...</p>}>
      {/* HeavyChart.js is fetched from network here */}
      {/* While it loads → fallback is shown */}
      {/* Once loaded → HeavyChart renders */}
      <HeavyChart />
    </Suspense>
  );
}

// You can also nest Suspense for granular loading states:
function Page() {
  return (
    <Suspense fallback={<p>Loading page...</p>}>
      <Suspense fallback={<p>Loading sidebar...</p>}>
        <Sidebar />
      </Suspense>
      <MainContent />
    </Suspense>
  );
}
07 Avoiding Race Conditions in Data Fetching Async
You search "cat", then quickly change it to "dog". The "cat" response arrives after the "dog" response and overwrites it — now you see cat results for a dog search. That's a race condition. Fix: ignore responses from old/cancelled requests. 
import { useState, useEffect } from 'react';

function Search({ query }) {
  const [data, setData] = useState(null);

  useEffect(() => {
    let cancelled = false;   // 🔑 flag for THIS specific request

    fetch(`/api/search?q=${query}`)
      .then(res => res.json())
      .then(result => {
        if (!cancelled) {       // only update if this request is still relevant
          setData(result);
        }
        // if cancelled=true, we silently discard the stale response
      });

    // Cleanup: runs when query changes OR component unmounts
    return () => { cancelled = true; };

  }, [query]);  // re-runs every time query changes

  return <div>{data ? JSON.stringify(data) : 'Loading...'}</div>;
}

// AbortController is the cleaner native alternative:
useEffect(() => {
  const controller = new AbortController();
  fetch(url, { signal: controller.signal })
    .then(res => res.json()).then(setData);
  return () => controller.abort();  // cancels the in-flight HTTP request
}, [url]);
08 useRef & useCallback Hook
useRef — a box that holds a value (or a DOM node) that persists across renders without causing a re-render when it changes.
useCallback — memoizes a function so its reference doesn't change on every render, preventing unnecessary child re-renders.
useRef — two common uses
import { useRef } from 'react';

function Demo() {
  // USE 1: Access a DOM element directly
  const inputRef = useRef(null);
  const focusInput = () => inputRef.current.focus();  // directly manipulate DOM

  // USE 2: Store a value that survives re-renders (no re-render on change)
  const clickCount = useRef(0);
  const handleClick = () => {
    clickCount.current += 1;  // updates silently — doesn't re-render
    console.log(clickCount.current);
  };

  return (
    <div>
      <input ref={inputRef} />
      <button onClick={focusInput}>Focus</button>
      <button onClick={handleClick}>Count (silent)</button>
    </div>
  );
}
useCallback — stable function reference
import { useState, useCallback, memo } from 'react';

// memo = only re-renders if props change
const Button = memo(({ onClick, label }) => {
  console.log(`${label} rendered`);
  return <button onClick={onClick}>{label}</button>;
});

function Parent() {
  const [count, setCount] = useState(0);

  // ❌ Without useCallback: new function ref every render → Button re-renders every time
  // const handleSave = () => saveData();

  // ✅ With useCallback: same function ref → Button only re-renders if deps change
  const handleSave = useCallback(() => {
    saveData();
  }, []);  // [] = never recreated

  return (
    <div>
      <button onClick={() => setCount(c => c + 1)}>{count}</button>
      <Button onClick={handleSave} label="Save" />  {/* won't re-render on count change */}
    </div>
  );
}
09 Redux Toolkit State Management
Redux = a single global storage room for your app's state. Anyone can read from it. You change it only by sending named "actions".
Redux Toolkit is the modern, less-boilerplate version. 3 steps: slice → store → use in component. 
// STEP 1: Create a slice (a piece of the global state)
// counterSlice.js
import { createSlice } from '@reduxjs/toolkit';

const counterSlice = createSlice({
  name: 'counter',
  initialState: { value: 0 },
  reducers: {
    increment: (state) => { state.value += 1; },   // action → state change
    decrement: (state) => { state.value -= 1; },
    reset:     (state) => { state.value = 0; },
  },
});

export const { increment, decrement, reset } = counterSlice.actions;
export default counterSlice.reducer;
// STEP 2: Create the store
// store.js
import { configureStore } from '@reduxjs/toolkit';
import counterReducer from './counterSlice';

export const store = configureStore({
  reducer: {
    counter: counterReducer,     // state.counter is now available globally
  },
});
// STEP 3: Provide store + use in any component
// main.jsx
import { Provider } from 'react-redux';
import { store } from './store';

ReactDOM.render(
  <Provider store={store}><App /></Provider>,
  document.getElementById('root')
);

// Counter.jsx — any component, anywhere in the tree
import { useSelector, useDispatch } from 'react-redux';
import { increment, decrement, reset } from './counterSlice';

function Counter() {
  const count    = useSelector(state => state.counter.value); // READ
  const dispatch = useDispatch();                              // WRITE

  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={() => dispatch(increment())}>+</button>
      <button onClick={() => dispatch(decrement())}>-</button>
      <button onClick={() => dispatch(reset())}>Reset</button>
    </div>
  );
}
